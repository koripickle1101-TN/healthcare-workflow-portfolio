# Denial Root Cause Analysis Framework

**Project Type:** Self-directed learning project  
**Focus Area:** Denial prevention, revenue cycle analysis, workflow improvement  
**Brand System:** White background, black text, Vols Orange `#FF8200` accents

---

## Project Summary

This project focuses on analyzing healthcare claim denials as system signals rather than isolated billing issues. The goal is to demonstrate how denial patterns can be categorized, traced back to upstream workflow breakdowns, and addressed through prevention-focused improvements.

This is a simulated analytical framework created for portfolio development. It does not use real payer data, employer data, or protected health information.

---

## Problem Statement

Denials are often handled as one-off issues, requiring rework at the billing stage. However, many denials are repeatable and predictable when viewed as patterns.

Common challenges include:

- Lack of standardized denial categorization
- No linkage between denial and root workflow cause
- Repeated rework for the same issues
- Limited feedback loops to upstream teams
- Focus on correction rather than prevention

---

## Denial Categories

| Category | Description |
|---|---|
| Eligibility | Coverage inactive, incorrect payer, or benefit mismatch |
| Authorization | Required authorization missing or invalid |
| Documentation | Clinical documentation does not support billed service |
| Coding / Claim Error | Incorrect code, modifier, or claim structure |
| Timely Filing | Claim submitted outside payer deadline |
| Coordination of Benefits | Incorrect primary/secondary payer sequence |

## Root Cause Mapping

| Denial Type | Likely Root Cause | Workflow Stage |
|---|---|---|
| Eligibility Denial | Insurance not verified or incorrect | Patient Access / Intake |
| Authorization Denial | Requirement not confirmed | Authorization Workflow |
| Documentation Denial | Missing or incomplete notes | Clinical Documentation |
| Coding Denial | Coding mismatch or error | Coding / Billing |
| Timely Filing | Delay in submission | Billing Operations |

## Prevention Strategy

1. Standardize denial categories across the organization.
2. Track denial frequency by category.
3. Map each denial to a workflow stage.
4. Identify repeat patterns.
5. Implement upstream validation steps.
6. Create feedback loops to patient access, clinical, and billing teams.

---

## Workflow Feedback Loop

```text
Denial Identified
   ↓
Categorize Denial
   ↓
Map to Workflow Stage
   ↓
Identify Root Cause
   ↓
Implement Upstream Fix
   ↓
Monitor Repeat Denials
```

## RCM Insight

Denials are not random events.

They are signals that something in the workflow failed to validate information before submission.

Organizations that reduce denials effectively do not simply improve billing. They strengthen upstream processes.
